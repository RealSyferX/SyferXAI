# Fix Disk Full Issue (SQLITE_FULL Error)

## Problem
- Disk is 100% full (72G/72G)
- SQLite database at `/var/lib/9router/db/data.sqlite` is 1.1GB and reporting `SQLITE_FULL` errors
- 9router PM2 process returning 500 errors on API calls
- Old project backups in `/root/code/` consuming ~15GB

## Root Cause
1. SQLite database has grown to 1.1GB with accumulated usage logs, request details, usage history
2. Old project backups in `/root/code/` consuming ~15GB (uploads-backups, Nasdem_Web_V2.0, portfolio sites)
3. SQLite cannot write when disk is full → `SQLITE_FULL` errors → 500 responses

## Plan

### Phase 1: Free Disk Space (Immediate)
1. **Remove old project backups** in `/root/code/`:
   - `/root/code/uploads-backups` (~7.8GB)
   - `/root/code/Nasdem_Web_V2.0` (~7.6GB) 
   - `/root/code/portfolio-website` (~647MB)
   - `/root/code/portfolio-website-next` (~446MB)
   - Keep: `/root/code/Bybit_AutoTrade`, `/root/code/backend-screener-crypto`

2. **Clean SQLite database** (`/var/lib/9router/db/data.sqlite`):
   - Stop 9router PM2 process
   - Delete old `requestDetails` entries (keep last 7 days)
   - Delete old `usageHistory` entries (keep last 30 days)
   - Delete old `usageDaily` entries (keep last 90 days)
   - Run `VACUUM` to reclaim space
   - Restart 9router

### Phase 2: Verify Fix
1. Check disk space after cleanup
2. Restart 9router PM2 process
3. Test API endpoint with curl to verify 500 error is resolved

### Phase 3: Prevent Recurrence
1. Add SQLite auto-vacuum or scheduled cleanup job
2. Add log rotation for requestDetails.json (56MB) and growing)
3. Consider moving DB to separate volume or enabling WAL mode with checkpointing

## Commands to Execute

```bash
# 1. Stop 9router
pm2 stop 9router

# 2. Remove old project backups
rm -rf /root/code/uploads-backups /root/code/Nasdem_Web_V2.0 /root/code/portfolio-website /root/code/portfolio-website-next

# 3. Clean SQLite database (keep recent data only)
sqlite3 /var/lib/9router/db/data.sqlite "
  DELETE FROM requestDetails WHERE createdAt < datetime('now', '-7 days');
  DELETE FROM usageHistory WHERE timestamp < datetime('now', '-30 days');
  DELETE FROM usageDaily WHERE date < date('now', '-90 days');
  VACUUM;
"

# 4. Restart 9router
pm2 start 9router

# 5. Verify
df -h /
curl -X POST http://localhost:3003/api/v1/chat/completions -H "Authorization: Bearer sk-3f68432058f6317c-f5afxg-81892e14" -H "Content-Type: application/json" -d '{"model":"qd/qmodel_latest","messages":[{"role":"user","content":"test"}]}'
```

## Risk Assessment
- **Low risk**: Removing old project backups (appear unused)
- **Medium risk**: Deleting old DB records - keeping 7-90 days retention is safe
- **Mitigation**: DB backups exist in `/var/lib/9router/db/backups/`
